# 教学思路

## 课程建设目标

《智能航海》是一门面向本科生的全校选修课。课程目标聚焦基础认知和兴趣引导，帮助学生形成对未来航运科技的基本判断：一艘船要从“有人驾驶”走向“远程控制”和“自主航行”，需要哪些技术能力、运行条件和制度支撑。

课程以 MASS 技术挑战识别报告为研究底座，以《智能航海概论》教材草稿为知识框架，将原本分散的智能船舶、智能港口、人工智能、通信、法规和安全内容整合到一次完整航次中。

## 总体教学逻辑

### 1. 用航次建立兴趣

第一步先让学生看到问题本身：一艘未来自主船舶从南京港出发，经过长江、近海、远洋、关键海峡或运河，最后进入鹿特丹港。这个过程包含装货、离泊、航道航行、避碰、远洋通信、恶劣天气、进港、引航、靠泊和码头作业。

这样组织课程可以让学生直接感受到智能航海的复杂性。每一项技术都有明确场景，每一个案例都能落到航次中的某个环节。

### 2. 用挑战组织知识

课程不按技术清单堆叠内容，而按航次挑战组织知识。每周聚焦一个问题：

- 内河和港口场景下，系统如何识别小船、浮标、桥区和复杂交通流？
- 远洋航行中，通信链路不稳定时，远程控制如何保持安全？
- 自动避碰系统如何理解 COLREGs 和真实船员习惯？
- 进港阶段，船舶如何与 VTS、引航员、拖轮、码头和海关协同？
- 自主船舶的安全性如何被测试、认证和监管？

这些问题可以自然引出传感器、AI、通信、控制、数字孪生、网络安全和法规标准。

### 3. 用案例支撑概念

课程每个模块配一个可点击案例，帮助本科生建立直观印象：

- Yara Birkeland：短途电动自主集装箱船。
- MEGURI2040：日本多船型无人运航验证。
- Mayflower Autonomous Ship：无人远洋科研船。
- Sea Machines：自主控制和远程指挥系统。
- Orca AI、Avikus：AI 辅助航行和态势感知。
- Rotterdam Digital Port：面向未来自主物流的数字港口。

案例讲解时重点分析三个问题：项目解决什么问题、使用哪些技术、适合哪类运行场景。

### 4. 用四层能力控制难度

课程采用从易到难的能力递进：

1. **辅助决策**：系统看见、提醒、建议，船员负责判断。
2. **远程监控**：岸基人员接收船舶状态并提供支持。
3. **远程控制**：岸基人员能够在必要时接管船舶操作。
4. **自主航行**：系统在限定运行条件下完成感知、决策和控制。

这样可以避免一开始就讨论完全无人船带来的复杂责任问题。学生先理解自动化和辅助决策，再逐步进入远程控制和高度自主。

## 课堂组织建议

每次课建议采用“10分钟问题导入 + 45分钟概念讲解 + 25分钟案例演示 + 15分钟讨论”的结构。

| 环节 | 目的 | 示例 |
| --- | --- | --- |
| 问题导入 | 激发兴趣 | “如果船上没人，看到前方渔船怎么办？” |
| 概念讲解 | 建立基础 | MASS、自主等级、传感器、通信、避碰 |
| 案例演示 | 连接现实 | 打开项目官网、视频或港口页面 |
| 讨论 | 形成判断 | 让学生判断某项目属于哪一级自主能力 |

## 教学产出

课程主页建议形成四类稳定材料：

1. **主页简介**：给学生和外部访问者快速理解课程。
2. **内容架构**：展示课程层级和知识框架。
3. **航次挑战图谱**：把 MASS 技术挑战放入完整航线。
4. **课堂资源**：保留可点击演示链接，支持课堂分享。

后续配套教材完成后，可以将章节内容逐步对应到本课程的六类挑战中，使课程网站成为教材、课堂展示和学生作业之间的入口。
