# 内容架构

课程内容采用“一个航次、四层能力、六类挑战”的结构。主线来自 MASS 技术挑战识别报告，并与《智能航海概论》教材草稿衔接。

## 一条主线：南京到鹿特丹

课程用一艘未来自主船舶的完整航次组织知识：

南京港装货与离泊 → 长江内河航行 → 长江口与近海水域 → 远洋航行 → 关键海峡或运河 → 北海与鹿特丹港进港 → 自动化码头靠泊与作业。

这条主线的教学价值在于把技术放进场景：同一艘船在内河、近海、远洋、港口和国际监管环境中会遇到不同问题，学生可以看到智能航海技术的真实使用条件。

## 四层能力：从辅助到自主

参考 IMO 对 MASS 自主等级的划分，课程用四层能力帮助学生理解技术递进关系。

| 层级 | 课程表述 | 课堂关注点 |
| --- | --- | --- |
| L1 | 自动化与辅助决策 | 船员在船，系统提供监测、预警和建议 |
| L2 | 船上有人值守的远程控制 | 船岸通信、远程监控、人工接管 |
| L3 | 船上无人值守的远程控制 | 远程操作中心、冗余通信、应急响应 |
| L4 | 高度自主航行 | 环境理解、自主决策、系统验证和责任划分 |

## 六类挑战：课程核心框架

### 1. 航行环境感知

学生理解雷达、AIS、摄像机、红外、电子海图、气象海况和港口数据如何共同构成船舶的“看见世界”的能力。重点讨论复杂水域、小目标、恶劣天气、传感器误报漏报和多源融合。

### 2. 船岸通信与数据交换

学生理解自主船舶对通信连续性、带宽、时延、网络安全和数据完整性的要求。重点讨论内河、近海、远洋、港口和运河场景下通信条件的差异。

### 3. 航线规划与避碰决策

学生理解自主航行系统如何根据航线、交通流、COLREGs、气象海况和船舶操纵性能生成决策。重点讨论智能避碰、路径规划、风险评估和可解释决策。

### 4. 船舶控制与系统冗余

学生理解自主决策需要通过推进、操舵、动力管理、定位和自动控制系统落地。重点讨论执行机构可靠性、系统冗余、故障诊断和预测性维护。

### 5. 港口、航道与交通协同

学生理解自主船舶需要与港口、VTS、引航、拖轮、码头、海关和航运公司协同。重点讨论智能港口、自动化码头、数字港口、船岸协同和标准化数据接口。

### 6. 法规、责任与人才能力

学生理解 MASS 技术落地需要法规、认证、培训和责任体系共同支撑。重点讨论 IMO MASS Code、远程操作员能力、跨境航行合规、网络安全和课程改革。

## 与教材草稿的关系

《智能航海概论》教材草稿提供课程的基础框架：概念基础、发展现状、限制因素和发展路径。本课程主页进一步把这些内容压缩为本科选修课可用的教学结构：

1. **基础概念**：智能航海、智能船舶、智能港口、MASS。
2. **全球案例**：欧洲、日本、新加坡、英国、美国和中国相关实践。
3. **技术挑战**：感知、通信、决策、控制、协同、安全。
4. **航次应用**：南京到鹿特丹的分阶段挑战识别。
5. **未来路径**：从辅助决策到远程控制，再到高度自主航行。
