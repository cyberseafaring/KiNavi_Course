# 内容架构

课程内容采用“案例先行、航次贯穿、技术拆解、判断收束”的结构。全校选修课不把学生先带入大量专业定义，而是让学生先看到智能航海正在发生的真实变化，再逐步理解这些变化背后的工程逻辑。

## 一条主线：南京到鹿特丹

课程用一艘未来自主船舶的完整航次组织知识：

南京港装货与离泊 -> 长江内河航行 -> 长江口与近海水域 -> 远洋航行 -> 关键海峡或运河 -> 北海与鹿特丹港进港 -> 自动化码头靠泊与作业。

这条主线的价值在于把案例放回场景。同样是自主航行，港区离泊、内河避碰、近海交通、远洋通信和自动化码头协同面对的难题并不相同。学生需要逐步形成一种判断能力：一个项目为什么选择某条航线、某种船型、某个自动化等级，以及它还没有解决哪些更困难的场景。

## 四组案例：从项目进入技术

### 1. 固定航线与零排放物流

代表案例包括 Yara Birkeland 和 GENBU。课程重点讨论固定航线为什么适合早期落地，港口和船岸系统怎样降低运行不确定性，电动化、自动靠泊、船级认证和商业运营如何共同影响自主船舶的推广。

### 2. 多船型商用验证

代表案例包括 MEGURI2040、Olympia Dream Seto、Mikage 和 Hokuren Maru No.2。课程重点讨论日本项目如何把集装箱船、渡船和 Ro-Ro 船放进不同运营场景，为什么 Fleet Operation Center、稳定通信和多船监控成为社会实施的核心条件。

### 3. AI 船长与远洋无人科研

代表案例包括 Mayflower Autonomous Ship。课程重点讨论 AI Captain、边缘计算、无人科研任务和远洋环境感知，也用项目中的挑战说明海上无人系统必须面对设备可靠性、天气、能源、通信和远程监控问题。

### 4. 智能船桥与辅助操纵

代表案例包括 Orca AI、Avikus、Sea Machines、Massterly 和 OpenBridge。课程重点讨论智能航海更可能先进入船桥辅助、风险预警、智能瞭望、路径建议和远程监控，再逐步改变船舶操纵和值班方式。

## 六类问题：从案例拆回技术链

### 1. 航行环境感知

学生理解雷达、AIS、摄像机、红外、电子海图、气象海况和港口数据如何共同构成船舶的态势认知。课堂重点放在小目标、拥挤水域、恶劣天气、夜间航行、误报漏报和多源数据融合。

### 2. 船岸通信与远程支持

学生理解自主船舶对通信连续性、带宽、时延、网络安全和数据完整性的要求。课程会比较港区 5G、近海通信、卫星链路、移动 Fleet Operation Center 和通信中断时的降级运行。

### 3. 航线规划与避碰决策

学生理解系统如何根据交通态势、COLREGs、气象海况、船舶操纵性能和运行限制生成建议。课堂关注智能避碰、路径规划、风险排序和人能否理解机器建议。

### 4. 船舶控制与执行可靠性

学生理解自主决策需要通过推进、操舵、动力管理、定位和自动控制系统落地。课程讨论自动靠离泊、航迹保持、执行机构故障、冗余设计和人工接管。

### 5. 港口、航道与交通协同

学生理解自主船舶需要与 VTS、引航、拖轮、码头、船公司、监管机构和数字港口协同。课程用鹿特丹数字港口和 MASS Port Network 讨论港口如何为自主船舶准备数据服务和作业流程。

### 6. 法规、认证与社会接受

学生理解 MASS 从演示走向商业运营需要规则、认证、保险、培训和责任体系共同支撑。课程重点跟踪 IMO MASS Code、船级社认证、运行区域限制、远程操作员能力和公众信任。

## 课堂输出：每周都有可判断的问题

课程的小作业和展示围绕三个输出展开：

1. **案例卡片**：一个项目的船型、航线、任务、自动化等级、关键技术和运行状态。
2. **航次定位**：把案例放入南京到鹿特丹航线，判断它适合解决哪一段问题。
3. **边界分析**：指出案例没有解决的困难场景，例如恶劣天气、拥挤水域、通信中断、港口协同或责任划分。
