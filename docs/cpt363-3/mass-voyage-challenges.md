# MASS 航次挑战图谱

本页把 IAMU MASS 技术挑战识别报告中的核心思路转化为本科课堂可用的教学图谱。主线航次为“南京港到鹿特丹港”，阿姆斯特丹港可作为扩展讨论目的港。

## 航次阶段总览

| 阶段 | 场景 | 主要挑战 | 相关技术 |
| --- | --- | --- | --- |
| 1 | 南京港装货与离泊 | 码头协同、靠离泊、港区交通、作业安全 | 自动靠泊、港口调度、VTS、远程监控 |
| 2 | 长江内河航行 | 船流密集、小目标多、桥区和浅水限制 | 雷达/AIS/视觉融合、电子航道图、风险预警 |
| 3 | 长江口与近海 | 内河到海船交通规则变化，气象海况变化 | 航线重规划、海况感知、通信切换 |
| 4 | 远洋航行 | 长时间运行、设备健康、通信覆盖、极端天气 | 卫星通信、预测性维护、能效优化、数字孪生 |
| 5 | 关键海峡或运河 | 交通拥堵、规则约束、引航协同、安全风险 | 精准控制、VTS协同、远程操作、应急策略 |
| 6 | 北海与鹿特丹进港 | 港口数据交换、引航、拖轮、靠泊窗口 | 港口数字孪生、船岸数据交换、智能调度 |
| 7 | 自动化码头作业 | 岸桥、AGV、堆场与船舶计划衔接 | 自动化码头、物联网、5G、港口AI |

## 挑战一：环境感知

自主船舶需要持续理解周围环境。内河和港口场景中，系统要识别小船、渔船、浮标、桥墩、施工船和岸线；远洋场景中，系统要处理大风浪、低能见度、夜间目标和传感器遮挡。

课堂可引导学生观察：

- 雷达适合发现目标，但目标类型和意图需要进一步判断。
- AIS 能提供船名、航速和航向，但小船和异常目标可能没有有效信息。
- 摄像机和红外可以增强视觉理解，恶劣天气会降低可靠性。
- 多源融合可以提高稳定性，也会带来数据冲突和解释问题。

## 挑战二：通信连续性

MASS 从辅助决策走向远程控制后，船岸通信成为关键条件。航次从南京到鹿特丹会经过内河、近海、远洋、海峡、运河和港口，不同区域的网络条件差异明显。

课堂可引导学生讨论：

- 内河和港口可以更多依赖岸基网络、5G和专用通信。
- 远洋航行需要卫星通信和多通道备份。
- 远程控制需要关注时延、带宽、链路切换和数据完整性。
- 通信中断时，船舶需要具备降级运行和安全等待策略。

## 挑战三：自主决策

自主船舶需要在航线规划、避碰、能效、海况和法规之间做决策。课程不要求学生推导复杂算法，重点理解决策所需信息和约束条件。

课堂可引导学生分析：

- 避碰决策需要考虑 COLREGs，也要理解真实交通环境。
- 航线优化需要兼顾安全、时间、燃油和排放。
- 机器学习模型需要大量真实数据和可验证的安全边界。
- 高风险场景需要人工接管、远程确认或自动进入安全状态。

## 挑战四：控制与冗余

自主决策必须转化为船舶运动控制。推进、操舵、定位、动力、电力和自动控制系统需要长期稳定运行。

课堂可引导学生关注：

- 自动舵和航迹控制是智能航行的基础。
- 港口和运河场景需要更高精度的低速控制。
- 关键设备故障时，系统需要冗余和应急方案。
- 预测性维护可以降低远洋航行中的故障风险。

## 挑战五：船岸协同

一艘自主船舶无法独立完成全部航运活动。它需要与港口、VTS、引航、拖轮、船公司、海事监管、海关和码头系统持续协同。

课堂可引导学生思考：

- 港口需要提前知道自主船舶的到港计划、状态和操作需求。
- VTS 需要理解自主船舶的运行能力和限制。
- 远程操作中心需要清晰的人机界面和权限管理。
- 跨国航行需要统一的数据交换、认证和责任规则。

## 挑战六：法规与人才

MASS 的发展同时改变技术体系和人才结构。未来航海人才需要理解船舶、数据、自动化、法规和远程操作之间的关系。

课堂可引导学生讨论：

- IMO MASS Code 为什么需要覆盖风险评估、软件、连接性、远程操作和培训。
- 远程操作员与传统船员的能力要求有哪些差异。
- 自主系统发生事故时，船东、制造商、软件供应商和远程操作中心如何承担责任。
- 航海教育需要增加哪些智能航运、数据和系统安全内容。

## 一页式课程框架

```text
完整航次
  南京港 -> 长江 -> 近海 -> 远洋 -> 海峡/运河 -> 北海 -> 鹿特丹港

核心挑战
  感知      通信      决策      控制      协同      法规

关键技术
  雷达/AIS/视觉   卫星/5G/VDES   AI避碰/路径规划   自动舵/动力控制
  数字孪生/预测维护   远程操作中心   智能港口   MASS Code

教学目标
  看懂案例 -> 识别挑战 -> 解释技术 -> 讨论限制 -> 提出改进思路
```
