# 模块 7 人机协同、远程监督与指挥控制

本模块关注智能系统进入船桥后的人员组织变化。课程重点是值更官、舰船指挥人员、远程监督人员和智能系统之间如何分配职责，如何理解 AI 建议，如何设置接管条件和记录要求。

## 学习目标

- 理解 human-in-the-loop、human-on-the-loop 和 human-out-of-the-loop 的区别。
- 说明智能船桥、远程操作中心和指挥控制系统的基本信息需求。
- 分析 AI 建议的解释性、置信度、备选方案和风险提示。
- 能设计一份人机协同值更记录模板。

## 核心问题

1. AI 系统提供建议后，谁负责最终操纵判断？
2. 智能船桥应向值更官显示哪些信息？
3. 远程监督和本船操纵之间如何保持共同态势？
4. 何时需要人工接管，接管后如何记录和复盘？

## 人机协同模式

| 模式 | 课程解释 | 航海场景 |
| --- | --- | --- |
| Human-in-the-loop | 人员批准后系统执行 | 避碰行动建议、航线变更 |
| Human-on-the-loop | 系统执行，人员持续监督 | 航迹保持、自动瞭望、常规监控 |
| Human-out-of-the-loop | 系统在限定条件内自主处理 | 低风险稳定航段的局部自动执行 |
| Human override | 人员随时可终止或接管 | 感知冲突、异常会遇、设备故障 |

## 智能船桥信息需求

| 信息 | 作用 |
| --- | --- |
| 态势图 | 展示目标、航线、风险区域和传感器状态 |
| AI 建议 | 说明建议行动、理由、置信度和备选方案 |
| 运行条件 | 显示系统适用水域、天气、通信和传感器状态 |
| 接管提示 | 标出需要人员介入的触发条件 |
| 日志记录 | 保存输入、建议、人员判断、行动和复核结果 |

## 课堂训练

1. 设计一个智能船桥显示页面，标出必须显示的信息。
2. 给出 AI 避碰建议，让学员写出值更官询问系统的五个问题。
3. 模拟通信受限下的远程监督，判断哪些信息必须保留在本船。

## 复习问题

- 人机协同中“监督”与“接管”的区别是什么？
- AI 建议的解释性应包含哪些内容？
- 远程监督人员和本船值更官如何保持一致态势？
