# 课程安排

本安排按 16 周、32 学时设计，面向全校本科生选修课。每周以一个真实案例或一组项目进入，再回到“南京到鹿特丹”航次主线，形成案例观察、技术拆解和课堂判断。

| 周次 | 主题 | 案例入口 | 课堂判断 |
| --- | --- | --- | --- |
| 第1周 | 智能航海导论 | Yara Birkeland、GENBU、Mayflower | 为什么自主船舶先从固定航线、科研船和辅助瞭望开始？ |
| 第2周 | MASS 与自主等级 | IMO MASS Code、Maritime UK MASS Code | 自动化、远程控制和自主航行的边界在哪里？ |
| 第3周 | 南京到鹿特丹航次 | 南京港、长江、远洋、鹿特丹港 | 一次完整航次中哪些阶段最难自动化？ |
| 第4周 | 港口离泊与固定航线 | Yara Birkeland | 固定短途航线为什么适合早期自主航行？ |
| 第5周 | 内河与拥挤水域感知 | 长江、东京湾、濑户内海案例 | AI 需要看见哪些目标才能安全航行？ |
| 第6周 | AI 瞭望与船桥辅助 | Orca AI、Avikus | 智能系统怎样改变传统瞭望和值班？ |
| 第7周 | 船岸通信与远程监控 | Massterly、OpenBridge | 船上无人时，岸上的人必须看到什么？ |
| 第8周 | Fleet Operation Center | MEGURI2040 永久与移动 FOC | 一套岸基中心如何同时支持多艘船？ |
| 第9周 | 自动避碰与航线规划 | COLREGs 会遇场景、Sea Machines | 机器能否给出安全、合规、可解释的避碰建议？ |
| 第10周 | 控制执行与自动靠泊 | GENBU、Olympia Dream Seto | 算法决策如何变成舵角、车令和靠泊动作？ |
| 第11周 | 远洋无人科研 | Mayflower Autonomous Ship | 远洋无人船为什么容易暴露可靠性和能源问题？ |
| 第12周 | 智能港口与数字港口 | Port of Rotterdam、MASS Port Network | 港口需要怎样的数据服务才能接纳自主船舶？ |
| 第13周 | 网络安全与数据可信 | AIS 欺骗、GNSS 干扰、远程控制风险 | 如果输入数据错了，系统应当怎样降级？ |
| 第14周 | 认证、责任与社会接受 | ClassNK、MLIT、IMO MASS Code | 从演示到商用，谁来证明它足够安全？ |
| 第15周 | 项目展示 | 学生自选案例 | 这个项目解决了哪一段航次挑战，还缺什么？ |
| 第16周 | 课程总结 | 四组案例回看 | 智能航海未来五年最可能先改变哪些岗位和流程？ |

## 课程小论文建议题目

1. 从 Yara Birkeland 看固定航线自主船舶的落地条件。
2. MEGURI2040 对多船型商用验证的启示。
3. Fleet Operation Center 如何改变船舶值班和岸基支持。
4. AI 瞭望系统在船桥辅助决策中的作用与限制。
5. Mayflower Autonomous Ship 的远洋无人航行经验与风险。
6. 鹿特丹数字港口如何支持未来自主船舶进港。
7. IMO MASS Code 对智能航海教育和船员能力的影响。

## 平时讨论建议

- 每次课保留一个“案例打开”环节，优先使用 [课堂资源](resources.md) 中的官方项目页面。
- 每名学生选择一个项目长期跟踪，记录其船型、航线、技术方案、运行状态和争议点。
- 分组展示重点说明“项目解决了哪一类航次挑战”，并明确它尚未覆盖的运行条件。
