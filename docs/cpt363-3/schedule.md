# 课程安排

本安排按 16 周、32 学时设计。每周围绕一个可讨论的问题展开，适合本科生选修课使用。

| 周次 | 主题 | 关键问题 | 课堂活动 |
| --- | --- | --- | --- |
| 第1周 | 智能航海导论 | 未来船舶为什么需要“智能”？ | 观看典型自主船舶项目，建立课程地图 |
| 第2周 | MASS 与自主等级 | 自动化、远程控制、自主航行有什么区别？ | 用 IMO 四级框架分析案例 |
| 第3周 | 南京到鹿特丹航次 | 一次完整远洋航次包含哪些阶段？ | 绘制航线与阶段任务 |
| 第4周 | 内河与港口离泊 | 在长江和港区，自主船舶最难识别什么？ | 分析拥挤水域、小目标和VTS协同 |
| 第5周 | 近海与复杂交通流 | 船舶如何感知周围目标和风险？ | 讨论雷达、AIS、视觉和电子海图 |
| 第6周 | 远洋通信 | 船岸数据链断续时系统怎样运行？ | 比较卫星、蜂窝、VDES和岸基网络 |
| 第7周 | 气象海况与航线优化 | 航线规划怎样兼顾安全、时间和能耗？ | 以台风、风浪和燃油消耗为案例 |
| 第8周 | 自动避碰决策 | 机器能否理解“让路船”和“直航船”？ | 用典型会遇场景讨论 COLREGs |
| 第9周 | 船舶控制与执行 | 决策如何转化为舵角、车钟和航迹？ | 认识自动舵、动力系统和定位控制 |
| 第10周 | 远程操作中心 | 岸上的人如何监控一支自主船队？ | 分析 Massterly、MEGURI2040 等项目 |
| 第11周 | 进港、引航与自动靠泊 | 港口阶段为什么比远洋更复杂？ | 讨论拖轮、引航、码头和自动化装卸 |
| 第12周 | 智能港口与数字港口 | 港口如何为自主船舶提供数据服务？ | 查看鹿特丹数字港口案例 |
| 第13周 | 网络安全与系统可靠性 | 自主船舶如何防止数据错误和网络攻击？ | 建立风险清单和应急策略 |
| 第14周 | 法规、认证与责任 | 自主船出了问题，责任如何划分？ | 阅读 IMO MASS Code 最新动态 |
| 第15周 | 技术路线与未来职业 | 从 L1 到 L4 需要哪些技术和人才？ | 学生分组展示项目案例 |
| 第16周 | 课程总结 | 一艘自主船完成航次还缺什么？ | 课程小论文答疑与总结 |

## 课程小论文建议题目

1. 从南京到鹿特丹航次看 MASS 的关键技术挑战。
2. 自主船舶在港口进出阶段的技术难点与解决思路。
3. 船岸通信对远程控制船舶的影响。
4. AI 视觉技术在船舶避碰中的应用与限制。
5. 智能港口如何支持自主船舶商业化运行。
6. IMO MASS Code 对未来航海教育的影响。

## 平时讨论建议

- 每次课保留一个“点击演示”环节，使用 [课堂资源](cpt363-3/resources.md) 中的官方项目或企业页面。
- 每名学生选择一个项目长期跟踪，如 Yara Birkeland、MEGURI2040、Mayflower、Sea Machines、Orca AI、Avikus 或 Rotterdam Digital Port。
- 分组展示时重点说明“项目解决了哪一类航次挑战”，避免只介绍公司背景。
