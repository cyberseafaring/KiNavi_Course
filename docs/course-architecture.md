# KiNavi 课程层级架构梳理

本文先把三门课的内容层级、首页短描述、资源保留边界和后续整理顺序梳理清楚。目标是让课程主页适合学生快速理解、适合对外分享，也便于后续逐页清理旧资料。

## 0. 总体判断

当前站点的主要问题集中在三点：

1. 课程中心页描述过长，三门课的差异被长段落稀释。
2. `cpt363-1` 和 `cpt363-3` 仍残留较多 UX 课程模板、Canvas 链接、占位模块和旧周次信息。
3. `cpt363-2 / 船联网` 已经有较完整的教材资源平台，但资源入口较多，学生容易分不清主线、课堂资源、实验资源和扩展资源。

推荐采用统一的四层信息架构：

```text
课程中心页
  -> 单门课程主页
    -> 核心学习主线
      -> 支撑资源库
```

每门课主页只保留 4 类信息：

- 一句话定位
- 3 个核心学习问题
- 推荐学习路径
- 主要资源入口

## 1. 三门课定位

| 课程 | 站点目录 | 当前定位 | 首页表达重点 |
|---|---|---|---|
| 智能航海 | `cpt363-3` | 选修课 / 兴趣引导 / 前沿认知 | 激发兴趣，建立智能航运全景，鼓励学生用 AI、数据和案例继续探索 |
| 船舶导航雷达 | `cpt363-1` | 专业核心技能课 / 设备操作与安全判断 | 雷达原理、目标识别、避碰判断、复杂环境下的安全使用 |
| 船联网 | `cpt363-2` | 教材配套课 / 系统工程与数据实践 | 感知、通信、数据治理、安全和应用的完整技术链 |

## 2. 课程中心页推荐结构

课程中心页建议保持短、清晰、可分享。每门课控制在 2 到 3 句话。

### 2.1 智能航海

**短描述**

面向对智能船舶、智慧港口、AI 与航运数字化感兴趣的学生。课程用案例、论文、工具和小型实践帮助学生看懂智能航运正在发生什么，并找到可以继续探索的方向。

**核心标签**

- 兴趣驱动
- 智能航运全景
- AI 与数据应用
- 案例阅读与创新实践

**主页重点**

- 课程是选修课，首页语气应轻一点，重点回答“为什么值得选”。
- 内容先提供一张智能航运地图，再逐步展开 14 个章节细节。
- 适合增加“你会看到什么、能尝试什么、最后能讲清什么”这样的入口。

### 2.2 船舶导航雷达

**短描述**

围绕船舶雷达的原理、操作、目标跟踪、避碰和维护展开。课程重点训练学生用雷达理解海上态势，在受限水域、能见度受限和复杂交通环境中作出安全判断。

**核心标签**

- 船舶导航安全
- 雷达原理与操作
- 目标跟踪与 ARPA
- 避碰与态势判断

**主页重点**

- 课程是专业技能导向，首页要突出“看懂雷达图像并用于安全决策”。
- 课程描述应回到学生能掌握的设备能力和安全判断。
- 资源入口应优先呈现教材、课件、周次安排和实操任务。

### 2.3 船联网

**短描述**

以《船联网技术基础》教材为主线，串联感知、通信、数据治理、安全和典型应用。课程配套章节资源、16 节范例课程、8 个实验项目、数据样本和交互测验。

**核心标签**

- 船联网系统架构
- 船端感知与海上通信
- 数据治理与安全
- 实验、数据和交互资源

**主页重点**

- 首页要明确“教材 8 章是主线”，静态资源平台是配套资源库。
- 资源适合按层级进入，首页只保留主入口。
- 需要把资源分成“必学、课堂、实践、扩展、归档”。

## 3. 推荐站点层级

### 3.1 总入口

```text
docs/home.md
  - 智能航海
  - 船舶导航雷达
  - 船联网
  - 课程层级架构
```

总入口负责分流，完整课程说明放到各课程主页。

### 3.2 单门课程主页

每门课主页建议统一为以下结构：

```text
课程名称
  1. 一句话定位
  2. 这门课解决的 3 个问题
  3. 学习路径
  4. 核心资源入口
  5. 适合人群
```

示例：

```text
智能航海
  - 为什么智能航运正在改变行业？
  - AI、物联网、大数据如何进入船舶、港口和水运管理？
  - 学生能用哪些工具做一个小型探索？
```

```text
船舶导航雷达
  - 雷达如何探测目标？
  - 驾驶员如何读图、跟踪和避碰？
  - 设备在复杂天气和复杂交通环境下如何可靠使用？
```

```text
船联网
  - 船端数据从哪里来？
  - 海上链路如何传输和降级？
  - 数据如何治理、安全共享并服务典型应用？
```

## 4. 船联网课程重点重构

`cpt363-2` 是当前资源量最大的课程，应使用“教材主线 + 资源库”的架构。

### 4.1 船联网主线

```text
船联网
  第1章 物联网技术基础
  第2章 船联网技术演变
  第3章 船联网体系架构
  第4章 船载感知信息采集
  第5章 海上通信网络基础
  第6章 船联网数据治理
  第7章 船联网安全机制
  第8章 典型应用发展趋势
```

这 8 章应作为唯一主骨架。`module-01` 到 `module-08` 对应 8 章，适合保留在侧边栏主导航。

### 4.2 船联网资源分层

| 层级 | 资源 | 处理建议 |
|---|---|---|
| 必学主线 | `topics.md`、`schedule.md`、`module-01` 到 `module-08` | 保留为学生第一入口 |
| 课堂支撑 | `lectures/L01` 到 `lectures/L16` | 每章 2 节课时，挂到对应章节 |
| 实践训练 | `labs/lab01` 到 `labs/lab08`、`datasets/*` | 按章节绑定，不在首页堆满 |
| 快速检查 | `interactive/quizzes`、`interactive/diagrams`、`interactive/simulations` | 用于课后复习和课堂互动 |
| 扩展学习 | `tutorials/python-basics`、`raspberry-pi`、`computer-vision`、`maritime-data` | 放到“进阶学习”，避免干扰主线 |
| 归档/弱化 | 旧 UX 页面、旧 LMS 入口、无课程关系的外部模板 | 从主导航移出 |

### 4.3 船联网推荐导航

```text
cpt363-2/home.md
  - 课程说明
  - 教材地图
  - 16 节课时安排
  - 配套数字资源

cpt363-2/topics.md
  - 8 章教材地图
  - 每章对应模块
  - 每章对应课时、实验、数据

cpt363-2/resources.md
  - 章节资源
  - 范例课程
  - 实验项目
  - 数据资源
  - 交互内容
  - 扩展教程
```

### 4.4 船联网章节到资源映射

| 章节 | 核心问题 | 课堂资源 | 实践资源 |
|---|---|---|---|

### 4.5 船联网需要弱化的内容

- `cpt363-2/lms-schedule.md`、`cpt363-2/lms-topics.md`：可保留兼容说明，从课程入口移出。
- 旧 Canvas 作业、测验和讨论链接：若没有当前课程对应权限，建议从对外页面移除。
- `module-09` 到 `module-13`：建议作为高级实践或课程项目，放到二级导航或项目专区。

## 5. 智能航海推荐重构

### 5.1 推荐主线

```text
智能航海
  1. 智能航运概论
  2. AI、物联网、大数据基础
  3. 智能船舶
  4. 智能导航与避碰
```

- 为什么选这门课

## 6. 船舶导航雷达推荐重构

`cpt363-1` 有较完整的雷达 `schedule.md`、`module-01`、`module-02` 和教材资源，但后续模块、主题和资源中混入 UX 模板。

### 6.1 推荐主线

```text
船舶导航雷达
  1. 雷达基础与船用雷达发展
  2. 系统组成与关键参数
  3. 电磁波传播、目标回波和干扰
  4. 雷达显示、操作和图像调节
  5. 雷达定位与导航
  6. 目标跟踪、ARPA 和避碰
  7. 维护、测试和故障处理
  8. 新型雷达、图像处理和数据融合
```

### 6.2 首页建议

船舶导航雷达首页应围绕“安全使用雷达”展开：

- 看懂雷达图像
- 正确设置和调节雷达
- 识别目标和环境干扰
- 用 ARPA、CPA、TCPA 支持避碰判断
- 进行基本维护和故障识别

### 6.3 应归档或移出主导航

- `cpt363-1/resources.md`：当前主要是 UX 资源。
- `cpt363-1/topics.md`：很多主题可作为选题库，但需要重新筛选。
- `cpt363-1/module-03` 到 `module-05`：主要是 UX 课程。
- `cpt363-1/module-06` 到 `module-13`：多为占位模板。
- `cpt363-1/lms-schedule.md` 和 `lms-topics.md`：旧 UX LMS 内容。
- `cpt363-1/_sidebar.md` 中 `No agenda` 和旧 Canvas 链接：建议移出对外导航。

## 8. 后续执行顺序

建议分三步推进，先清导航，再清首页，最后清历史资料。

1. **总入口压缩**：已将 `docs/home.md` 改为短描述，并加入本架构页。
2. **导航清理**：移除三门课侧边栏和导航栏中的 UX、Canvas 旧入口、占位模块和过期日期。
3. **课程主页重写**：按“定位、核心问题、学习路径、资源入口”重写 `cpt363-1/home.md`、`cpt363-2/home.md`、`cpt363-3/home.md`。
4. **资源归档**：把旧 UX 文件和占位模块移到 `archive` 或从导航中移除。
5. **船联网资源矩阵固化**：在 `cpt363-2/topics.md` 或 `resources.md` 中固定章节、课时、实验、数据、测验的对应关系。

## 9. 当前保留规则

- 学生第一眼看到的页面只保留核心课程信息。
- 教材、课件、实验、数据、测验保留。
- 与当前三门课无关的 UX 模板资源移出导航。
- 旧 LMS 链接只有在仍能访问且对应当前课程时保留。
- 无内容的占位模块从导航中移出。
