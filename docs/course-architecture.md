# KiNavi 课程层级架构梳理

本文先把四门课的内容层级、首页短描述、资源保留边界和后续整理顺序梳理清楚。目标是让课程主页适合学生快速理解、适合对外分享，也便于后续逐页清理旧资料。

## 0. 总体判断

当前站点的主要问题集中在三点：

1. 课程中心页描述过长，课程之间的差异容易被长段落稀释。
2. `cpt363-1` 仍残留较多 UX 课程模板、Canvas 链接、占位模块和旧周次信息。
3. `cpt363-2 / 船联网` 已经有较完整的教材资源平台，但资源入口较多，学生容易分不清主线、课堂资源、实验资源和扩展资源。

推荐采用统一的四层信息架构：

```text
课程中心页
  -> 单门课程主页
    -> 核心学习主线
      -> 支撑资源库
```

每门课主页只保留 4 类信息：

- 一句话定位
- 3 个核心学习问题
- 推荐学习路径
- 主要资源入口

## 2. 课程中心页推荐结构

课程中心页建议保持短、清晰、可分享。每门课控制在 2 到 3 句话。

### 2.1 智能航海（全校选修课）

**短描述**

面向对智能船舶、AI 与未来航运感兴趣的全校本科生。课程以 MASS 技术挑战识别为主线，用“南京到鹿特丹”的完整航次串联环境感知、船岸通信、自主决策、远程控制、智能港口和法规标准。

**核心标签**

- 兴趣驱动
- MASS 技术挑战识别
- 南京到鹿特丹完整航次
- 课堂案例与点击演示资源

**主页重点**

- 课程是全校选修课，首页应重点回答“为什么值得选”。
- 内容先提供一张完整航次地图，再展开感知、通信、决策、控制、协同和法规六类挑战。
- 资源页应保留可点击演示链接，帮助教师把课堂讲解和国际项目现场连接起来。

### 2.2 智能航海（海军专业课）

**短描述**

在全校选修课基础上深化，面向海军军官院校开设，32 学时。课程从人工智能基础与前沿知识切入，沿“感知、理解、预测、规划、决策、控制、复核”主线解释 AI 如何改变舰船操纵和值更训练。

**核心标签**

- 海军军官院校
- AI 基础与前沿 AI
- 感知到控制
- 舰船操纵与人机协同

**主页重点**

- 课程对象、32 学时结构和军官能力目标必须清楚。
- 内容先建立 AI 知识线，再展开感知、预测、规划、避碰、控制和复核。
- 资源页保留 IMO、可信 AI 和智能航运项目链接，帮助教师把制度背景、技术案例和操纵训练连接起来。

### 2.3 船舶导航雷达

**短描述**

围绕船舶雷达的原理、操作、目标跟踪、避碰和维护展开。课程重点训练学生用雷达理解海上态势，在受限水域、能见度受限和复杂交通环境中作出安全判断。

**核心标签**

- 船舶导航安全
- 雷达原理与操作
- 目标跟踪与 ARPA
- 避碰与态势判断

**主页重点**

- 课程是专业技能导向，首页要突出“看懂雷达图像并用于安全决策”。
- 课程描述应回到学生能掌握的设备能力和安全判断。
- 资源入口应优先呈现教材、课件、周次安排和实操任务。

### 2.4 船联网

**短描述**

以《物联网与船联网技术》教材为主线，串联感知、通信、数据治理、安全和典型应用。课程配套章节资源、16 节范例课程、8 个实验项目、数据样本和交互测验。

**核心标签**

- 船联网系统架构
- 船端感知与海上通信
- 数据治理与安全
- 实验、数据和交互资源

**主页重点**

- 首页要明确“教材 8 章是主线”，静态资源平台是配套资源库。
- 资源适合按层级进入，首页只保留主入口。
- 需要把资源分成“必学、课堂、实践、扩展、归档”。

## 3. 推荐站点层级

### 3.1 总入口

```text
docs/home.md
  - 智能航海
  - 船舶导航雷达
  - 船联网
  - 课程层级架构
```

总入口负责分流，完整课程说明放到各课程主页。

### 3.2 单门课程主页

每门课主页建议统一为以下结构：

```text
课程名称
  1. 一句话定位
  2. 这门课解决的 3 个问题
  3. 学习路径
  4. 核心资源入口
  5. 适合人群
```

示例：

```text
智能航海
  - 为什么智能航运正在改变行业？
  - AI、物联网、大数据如何进入船舶、港口和水运管理？
  - 学生能用哪些工具做一个小型探索？
```

```text
船舶导航雷达
  - 雷达如何探测目标？
  - 驾驶员如何读图、跟踪和避碰？
  - 设备在复杂天气和复杂交通环境下如何可靠使用？
```

```text
船联网
  - 船端数据从哪里来？
  - 海上链路如何传输和降级？
  - 数据如何治理、安全共享并服务典型应用？
```

## 4. 船联网课程重点重构

`cpt363-2` 是当前资源量最大的课程，应使用“教材主线 + 资源库”的架构。

### 4.1 船联网主线

```text
船联网
  第1章 物联网技术基础
  第2章 船联网技术演变
  第3章 船联网体系架构
  第4章 船载感知信息采集
```

## 5. 智能航海双轨结构

`cpt363-3` 保留为面向全校本科生的《智能航海》选修课，使用 MASS 航次挑战和国际示范项目建立兴趣入口。`cpt363-4` 是在选修课基础上深化形成的海军专业课，面向海军军官院校，重心转向 AI 基础、前沿 AI 与舰船操纵训练。

### 5.1 全校选修课主线

```text
智能航海（全校选修课）
  1. 课程主页：全校选修课定位、兴趣入口和学习收获
  2. 教学思路：从 MASS 报告到本科课堂的组织方式
  3. 内容架构：一个航次、四层能力、六类挑战
  4. 航次挑战图谱：南京港到鹿特丹港的阶段化问题
  5. 课程安排：16 次课、32 学时的课堂节奏
  6. 课堂资源：规则、项目、公司、港口和技术演示链接
  7. 课程专题：小论文和分组展示选题
```

### 5.2 海军专业课主线

```text
智能航海（海军专业课）
  1. 课程主页：海军军官院校定位、32 学时结构和学习收获
  2. 教学思路：AI 概念如何进入舰船操纵训练
  3. 内容架构：从感知到控制的八个模块
  4. AI 与舰船操纵地图：AI 知识到操纵任务的对应关系
  5. 系统挑战图谱：数据、感知、预测、规划、避碰、控制和复核
  6. 课程安排：16 次课、32 学时的课堂节奏
  7. 课堂资源：IMO、可信 AI、智能航运项目和船桥设计案例
  8. 课程专题：课堂推演、综合演练和课程报告选题
```

### 5.3 首页建议

两门《智能航海》在首页上应清楚区分：

- 全校选修课强调兴趣、航次、案例和自主船舶技术全景。
- 海军专业课强调 AI 知识、舰船操纵、人机协同和安全接管。
- 两门课可以共享 MASS、智能港口和远程操作资源，但学习产出不同。

### 5.4 当前目录分工

- `cpt363-3`：全校选修课，保留 MASS 航次挑战、南京到鹿特丹主线和通识化案例讲解。
- `cpt363-4`：海军专业课，保留 AI 基础、前沿 AI、感知到控制、舰船操纵和人机协同训练。

后续维护时，不要把两门课合并。选修课作为兴趣入口和技术全景，专业课作为能力深化和操纵训练。

## 6. 船舶导航雷达推荐重构

`cpt363-1` 有较完整的雷达 `schedule.md`、`module-01`、`module-02` 和教材资源，但后续模块、主题和资源中混入 UX 模板。

### 6.1 推荐主线

```text
船舶导航雷达
  1. 雷达基础与船用雷达发展
  2. 系统组成与关键参数
  3. 电磁波传播、目标回波和干扰
  4. 雷达显示、操作和图像调节
  5. 雷达定位与导航
  6. 目标跟踪、ARPA 和避碰
  7. 维护、测试和故障处理
  8. 新型雷达、图像处理和数据融合
```

### 6.2 首页建议

船舶导航雷达首页应围绕“安全使用雷达”展开：

- 看懂雷达图像
- 正确设置和调节雷达
- 识别目标和环境干扰
- 用 ARPA、CPA、TCPA 支持避碰判断
- 进行基本维护和故障识别

### 6.3 应归档或移出主导航

- `cpt363-1/resources.md`：当前主要是 UX 资源。
- `cpt363-1/topics.md`：很多主题可作为选题库，但需要重新筛选。
- `cpt363-1/module-03` 到 `module-05`：主要是 UX 课程。
- `cpt363-1/module-06` 到 `module-13`：多为占位模板。
- `cpt363-1/lms-schedule.md` 和 `lms-topics.md`：旧 UX LMS 内容。
- `cpt363-1/_sidebar.md` 中 `No agenda` 和旧 Canvas 链接：建议移出对外导航。

## 8. 后续执行顺序

建议分三步推进，先清导航，再清首页，最后清历史资料。

1. **总入口压缩**：已将 `docs/home.md` 改为短描述，并加入本架构页。
2. **导航清理**：移除课程侧边栏和导航栏中的 UX、Canvas 旧入口、占位模块和过期日期。
3. **课程主页重写**：按“定位、核心问题、学习路径、资源入口”重写 `cpt363-1/home.md`、`cpt363-2/home.md`、`cpt363-3/home.md` 和 `cpt363-4/home.md`。
4. **资源归档**：把旧 UX 文件和占位模块移到 `archive` 或从导航中移除。
5. **船联网资源矩阵固化**：在 `cpt363-2/topics.md` 或 `resources.md` 中固定章节、课时、实验、数据、测验的对应关系。

- 无内容的占位模块从导航中移出。
