# 模块 8 安全验证、对抗环境与综合演练

本模块收束课程。重点是智能航行系统如何在异常、欺骗、通信受限、模型失效和设备故障条件下保持安全运行。课程以综合演练方式要求学员完成从感知到控制的闭环分析。

## 学习目标

- 理解可信 AI、风险管理、验证确认、运行条件与限制、安全降级和人工接管。
- 分析传感器欺骗、数据污染、通信受限和网络安全事件对航行安全的影响。
- 说明仿真测试、硬件在环、海试和训练演练之间的关系。
- 完成一份智能航行系统综合演练报告。

## 核心问题

1. 如何证明一个智能航行系统在指定条件下可安全使用？
2. 传感器欺骗或数据异常会怎样影响感知、决策和控制？
3. 系统何时应进入保守模式、安全降级或人工接管？
4. 军官院校如何训练学员识别 AI 系统边界？

## 安全验证框架

| 环节 | 目的 | 输出 |
| --- | --- | --- |
| 运行条件定义 | 明确系统适用水域、能见度、海况、通信和传感器条件 | ODD / 运行条件与限制 |
| 仿真测试 | 大规模覆盖常见和极端场景 | 场景通过率、失效模式 |
| 硬件在环 | 检查传感器、控制器和执行机构协同 | 实时性、延迟和故障响应 |
| 海试验证 | 在真实环境中检验系统表现 | 海试记录和限制修订 |
| 人员训练 | 训练解释、监督、接管和记录能力 | 值更记录和演练评估 |

## 对抗与异常场景

| 场景 | 影响 | 安全处置 |
| --- | --- | --- |
| GNSS 干扰或欺骗 | 位置和航迹估计错误 | 多源定位复核，降低自动化等级 |
| AIS 虚假或缺失 | 目标身份和运动意图错误 | 雷达、视觉和通信交叉验证 |
| 光电识别失败 | 夜间、雨雾、眩光下目标漏检 | 使用雷达和红外补充，增加人工瞭望 |
| 通信受限 | 远程监督延迟或中断 | 本船保守控制和预案执行 |
| 模型输出异常 | 建议不符合常识或规则 | 触发人工接管和日志记录 |
| 执行机构故障 | 舵、车、侧推或动力响应异常 | 切换冗余、降低航速、保持安全水域 |

## 综合演练

演练场景：一艘舰船在受限能见度条件下通过复杂水域，雷达发现多个目标，光电识别部分失效，AIS 信息存在延迟，AI 系统给出改向避碰建议，同时风流导致航迹偏移。

学员需要完成：

1. 标出感知输入和不确定性。
2. 判断 AI 风险评估是否可信。
3. 比较 AI 避碰建议与人工操纵方案。
4. 说明控制执行需要哪些状态和反馈。
5. 给出接管条件、安全降级和记录内容。

## 复习问题

- 可信 AI 在智能航海中包含哪些要求？
- 运行条件与限制如何帮助人员判断是否采纳 AI 建议？
- 一个完整的智能航行演练报告应覆盖哪些证据？
