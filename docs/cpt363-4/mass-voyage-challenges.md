# 智能航海（海军专业课）系统挑战图谱

本页把智能航海的技术挑战转化为军官院校课堂可用的训练图谱。图谱围绕舰船操纵任务展开，重点观察 AI 系统从数据输入到控制执行的每个环节如何改变值更官的判断方式、训练方式和安全责任。

## 从感知到控制的挑战总览

| 环节 | 关键问题 | AI 技术 | 操纵影响 | 军官训练重点 |
| --- | --- | --- | --- | --- |
| 数据输入 | 数据是否完整、可信、同步？ | 数据治理、时间同步、异常检测 | 错误输入会改变态势判断 | 识别数据缺失、延迟、冲突和异常来源 |
| 智能感知 | 目标、岸线、浮标、障碍物是否被正确识别？ | 目标检测、语义分割、多传感器融合 | 瞭望从人工观察扩展为人机共同确认 | 比对雷达、光电、AIS、电子海图和人工观察 |
| 态势理解 | 目标意图、交通流和航行限制是否清楚？ | 轨迹预测、场景理解、知识图谱 | 值更判断从单目标扩展到多目标态势评估 | 建立目标优先级、危险等级和规则约束 |
| 风险预测 | 风流浪、视程、目标机动和设备状态如何影响风险？ | 风险评估、数字孪生、概率模型 | 避碰提前量和安全裕度需要动态调整 | 解释风险变化原因，判断预警是否可信 |
| 路径规划 | 推荐航线是否满足规则、操纵性能和任务约束？ | 搜索规划、优化算法、强化学习 | 航线设计从人工经验扩展为算法辅助方案 | 审查规划依据、备选方案和不可行条件 |
| 避碰决策 | 避碰建议是否符合 COLREGs 与船舶实际操纵能力？ | 决策树、规则推理、学习型决策 | 避碰从口令式操纵扩展为建议、确认、执行闭环 | 复核让路/直航关系、CPA/TCPA 和操纵余量 |
| 控制执行 | 舵角、车令和航迹控制是否稳定可控？ | 控制算法、模型预测控制、自动舵 | 自动执行需要考虑船体惯性和环境扰动 | 观察执行偏差、振荡、超调和接管条件 |
| 人工复核 | 系统建议、告警和接管记录是否清晰？ | 可解释 AI、可信 AI、人机交互 | 人员职责转向监督、质疑、授权和复盘 | 形成接管决策、日志记录和责任链分析 |

## 挑战一：AI 基础知识如何转化为航海能力

人工智能基础知识需要落到航海任务上理解。数据、特征、模型、训练、验证和泛化等概念，分别对应船舶传感器数据、目标特征、风险模型、仿真训练、海试验证和陌生海况适应能力。学员需要理解模型输出的含义，也要理解模型输出的条件。

课堂训练可采用“同一目标，不同数据源”的方式展开：雷达给出回波，AIS 给出身份和运动信息，光电图像给出外形，电子海图给出水域限制，学员据此判断 AI 识别结果是否合理。

## 挑战二：前沿 AI 如何改变船舶操纵

深度学习提高了海上目标识别能力，强化学习推动复杂场景下的策略搜索，生成式 AI 可以辅助航行说明、训练复盘和任务问答，智能体技术正在把多步骤计划、工具调用和状态记忆结合起来。课程把这些前沿技术放进操纵任务中讨论：

- 深度学习对应目标识别、视觉增强、雷达图像理解和异常检测。
- 强化学习对应靠离泊、路径跟踪、避碰策略和仿真训练。
- 生成式 AI 对应值更问答、航行资料整理、演练复盘和解释生成。
- 智能体对应多源信息检索、计划草案生成、告警分派和训练辅助。

## 挑战三：多源感知如何进入值更判断

智能航海系统依赖雷达、AIS、光电、红外、GNSS、电子海图、气象海况和船体状态数据。多源融合可以提高态势完整性，也会产生冲突、延迟和不确定性。值更官需要掌握“互证”方法：用不同来源验证同一目标，用规则和环境解释异常，用人工观察补充机器感知盲区。

典型课堂问题包括：

- AIS 目标消失后，雷达回波是否仍然稳定？
- 光电识别为小艇的目标，在雷达上是否有连续运动轨迹？
- 电子海图显示的水深、禁航区和航道边界是否改变规划建议？
- 大雨、海杂波、夜间眩光和遮挡条件下，系统置信度是否下降？

## 挑战四：智能避碰如何接受规则与操纵性能约束

智能避碰需要同时处理 COLREGs、目标运动、海况、船舶惯性、航道限制和任务意图。课程强调学员对建议进行复核：系统给出的转向、减速或等待方案，需要说明规则依据、风险变化和执行余量。对于学习型决策，课堂重点关注训练场景、验证方法和失效条件。

避碰推演可采用四步：

1. 识别让路关系、直航义务和潜在会遇类型。
2. 计算或估计 CPA/TCPA、相对方位变化和目标机动趋势。
3. 比较 AI 推荐方案、传统规则判断和船舶实际操纵能力。
4. 给出确认、修正、拒绝或人工接管的理由。

## 挑战五：控制执行如何与人工接管衔接

AI 规划结果需要通过操舵、推进、定位和航迹控制落实到船舶运动。控制层需要面对风、流、浪、浅水效应、舵效变化和执行机构延迟。学员需要理解自动控制的表现形式：航迹偏差、横向误差、艏向超调、速度响应、低速漂移和靠泊余量。

接管条件建议从四类信号判断：

- 感知异常：目标丢失、分类冲突、定位跳变、置信度骤降。
- 决策异常：推荐动作与 COLREGs、航道限制或任务意图冲突。
- 控制异常：航迹误差持续增大、舵令频繁振荡、执行偏差超限。
- 协同异常：通信中断、远程授权失败、告警未确认、日志缺失。

## 挑战六：可信 AI 与制度背景

智能航海课程需要引入可信 AI、风险管理和 MASS Code 的制度背景。NIST AI 风险管理框架强调治理、映射、度量和管理，OECD AI 原则强调以人为本、透明、稳健、安全和问责。IMO 正在推进 MASS Code，这为自主船舶风险评估、运行条件、软件原则、连接性、远程操作、培训和值班提供了重要背景。

在军官院校课程中，这些制度要求应转化为训练问题：

- 系统建议是否能说明数据来源和推理依据？
- 风险等级变化是否被记录、复核和解释？
- 远程监督与船上值更之间如何分配权限？
- 模型失效、通信受限和传感器异常时，安全状态如何定义？

## 一页式课程框架

```text
AI 知识
  数据 -> 模型 -> 训练 -> 评估 -> 深度学习 -> 强化学习 -> 生成式 AI -> 可信 AI

操纵任务
  瞭望 -> 定位 -> 态势判断 -> 避碰 -> 路径规划 -> 航迹控制 -> 人工接管 -> 复盘

系统链路
  感知 -> 理解 -> 预测 -> 规划 -> 决策 -> 控制 -> 告警 -> 人工复核

军官能力
  看懂系统 -> 复核建议 -> 组织协同 -> 掌握接管 -> 记录证据 -> 改进训练
```
