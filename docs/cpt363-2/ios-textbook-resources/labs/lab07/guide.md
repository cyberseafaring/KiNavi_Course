# 实验目标

本实验完成一次简化的多源融合流程，目标是让学生掌握时间对齐、目标关联与态势生成的基本步骤，并理解不确定性如何在工程上表现出来。

## 数据与工具

- AIS 解析数据：`../../datasets/ais-samples/ais_decoded_sample.csv`
- 轨迹样例：`../../datasets/ais-samples/vessel_tracks.csv`
- Python 3.8+
- 建议库：`pandas`、`numpy`

## 实验步骤

## 时间对齐

对齐不同数据源的时间轴，选择一种对齐策略并说明理由。对齐策略需要明确插值方式或最近邻方式，并记录对齐误差。

## 目标关联

在简化条件下实现关联规则，例如基于距离门限与速度约束的关联。关联结果需要输出“关联成功率”和“疑似冲突样例”，避免只给出最终轨迹而不解释过程。

## 融合输出

输出融合后的轨迹或态势结果，并给出质量标注字段，说明哪些点来自高质量数据，哪些点来自推断或插值。

## 预期结果

- 一条或多条融合轨迹
- 一份包含质量标注的融合结果表
- 一段对关联错误的分析说明
