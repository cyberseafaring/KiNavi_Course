# 实验目标

本实验以“可控、可复现、可回滚”为原则完成一次安全威胁模拟与防护建议输出。实验不包含真实攻击与破坏性操作，重点是让学生学会用日志与证据链定位问题并提出可执行的防护策略。

## 安全边界

实验仅在本地或隔离环境中进行，不扫描外网目标，不进行未授权访问，不执行破坏性载荷。所有操作以“复现风险条件”和“验证防护效果”为目的。

## 数据与工具

- 示例日志：`../../datasets/security/security_log_sample.csv`
- Python 3.8+

## 实验步骤

## 场景设定

选择一个安全场景，例如弱口令登录、异常请求频率或配置错误导致的越权访问。场景设定要明确资产、攻击面和预期影响。

## 日志分析

基于示例日志提取关键字段，定义最小检测规则，例如同一来源的高频失败登录、异常路径访问或时间窗口内的异常突增。规则需要写清楚触发条件，避免只给出“看起来异常”的描述。

## 防护策略

针对场景给出防护策略，至少包含：

- 身份与权限策略建议
- 日志与审计建议
- 弱网条件下的降级原则

## 预期结果

- 一份简短的分析报告，包含证据链与结论
- 一组可执行的防护建议，说明适用条件与代价
