L11

船联网数据解析存储

第6章 · 实验演示课 · 45分钟

教学目标

本节课通过解析示例报文，把数据治理链条从抽象概念落到可操作的工程步骤。

  • 能够区分数据类型分层与时间语义
  • 能够完成简单 NMEA 语句解析与字段校验
  • 能够理解原始层、清洗层、产品层的边界

内容要点

  • 数据类型分层与存储策略
  • NMEA 语句结构与校验
  • 解析结果的字段命名与版本管理意识

课堂节奏

  • 演示：读取样本日志并解析字段
  • 讲解：时间戳与存储分层
  • 练习：为解析结果设计最小数据字典

教材要点

  • 解析不是终点，解析结果必须连同来源、时间语义、质量标注和版本信息一起进入治理链条
  • 原始层、清洗层、产品层要分开保存，保证排错、复盘和共享时都能找到责任边界
  • 采集时间优先于到达时间，到达时间和处理时间主要用于解释延迟、补传和排队问题
  • 字段命名、单位和编码规则必须进入数据字典，否则跨船端、跨版本会出现口径漂移

实验任务

  • 任务1：读取 AIS 或 NMEA 样本，拆出字段、单位、校验和解析状态
  • 任务2：为解析结果设计最小数据字典，至少包含字段名、含义、单位、时间语义和版本号
  • 任务3：把原始记录、清洗记录和产品记录各写一条，比较它们应保留哪些信息
  • 任务4：选一个异常样本，说明它应该在解析阶段报错、在清洗阶段标注，还是在产品层降级

讨论框架

  • 先讨论“为什么不能只保留解析后的字段”，把追溯、排错和审计需求说清楚
  • 再讨论“时间语义错了会怎样”，用错序、补传、重复写入三个场景分析后果
  • 最后讨论“版本演进如何不停机”，把向后兼容、对账和过渡期映射作为重点

课后思考题

  1. 如果解析出的字段名称发生变更，如何保持历史数据可用。
  2. 为什么保存原始层数据对排错与追溯很重要。
