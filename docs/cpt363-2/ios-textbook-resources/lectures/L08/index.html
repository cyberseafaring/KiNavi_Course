L08

多源感知数据质量标注

第4章 · 实验演示课 · 45分钟

教学目标

本节课强调一个关键观点：融合之前先写清楚数据质量，能显著降低误判与误用风险。

  • 能够说明雷达、AIS、视觉等信息的互补关系
  • 能够理解缺测、延迟、异常等质量线索的表达方式
  • 能够概括降级策略的触发条件与结果形态

内容要点

  • 多源互补的工程动机与典型互补关系
  • 质量标注字段示例：缺测、延迟、异常、来源
  • 降级策略：关键信息优先与摘要模式

课堂节奏

  • 演示：同一目标在不同数据源中的差异表现
  • 讲解：质量标注如何进入数据结构
  • 练习：为一个数据源制定简单的质量规则

教材要点

  • 多源感知不是简单叠加，而是用 AIS、雷达、视觉等来源在覆盖范围、身份语义和环境适应性上互补
  • 质量标注要覆盖缺测、延迟、漂移、错配、来源和时间状态，避免“有数据”被误判为“数据可信”
  • 时间同步是质量标注的一部分，采集时间、到达时间、处理时间混用会直接放大融合误差
  • 降级策略要提前定义：哪些字段必须保留，哪些字段可以摘要化，哪些情况下需要触发告警或回退

实验任务

  • 任务1：读取 AIS 轨迹样本与视觉样本，列出各自能表达的目标信息和看不到的信息
  • 任务2：为一条观测记录设计最小质量字段集，至少包含 `source`、`sample_time`、`delay_state`、`confidence`
  • 任务3：模拟一个“视觉丢帧 + AIS 延迟”的场景，写出系统应如何降级显示与告警
  • 任务4：把任务结果扩展到 Lab06 或 Lab07，输出一页质量规则说明

讨论框架

  • 先讨论“缺测是否等于错误”，明确缺测、异常、不可判定三种状态的区别
  • 再讨论“质量字段给谁看”，区分算法内部使用、产品层展示和运维审计三种视角
  • 最后讨论“什么时候必须降级”，把链路受限、单源退化、多源冲突三种场景分别落地

课后思考题

  1. 为什么“缺测”需要显式表达，而不是简单留空。
  2. 在弱网条件下，质量标注字段是否需要降级，如何做。
