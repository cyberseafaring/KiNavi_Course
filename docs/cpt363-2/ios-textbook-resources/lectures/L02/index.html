L02

物联网核心技术栈

第1章 · 知识点讲解 · 45分钟

教学目标

本节课把“系统如何工作”落实到可操作的技术栈视角，为后续船联网分层架构打基础。

  • 能够说明从物理量到数字信号再到信息产品的转换路径
  • 能够解释云计算与边缘计算的分工逻辑
  • 能够用“业务需求驱动选型”的方法理解技术取舍

内容要点

  • 传感器与采集链路，强调量程、精度与采样率的含义
  • 连接层常见组织方式，强调消息结构与弱网策略
  • 云边协同的部署位置判断，强调实时性与断链可用
  • 向船联网迁移时的关键变化，强调海上通信约束

课堂节奏

  • 导入：用一个简单传感器示例复盘技术链条
  • 讲解：传感器、边缘计算、云服务的职责边界
  • 讨论：把同一业务分别放在船端与岸端的利弊

资源清单

  • 教学PPT：待补充
  • 精讲视频：待补充
  • 实验指引：建议配合 Lab01 与 Lab02

课后思考题

  1. 同一条业务链路中，哪些信息适合在边缘侧预处理，原因是什么。
  2. 如果链路带宽受限，你会优先压缩哪一类数据，为什么。
