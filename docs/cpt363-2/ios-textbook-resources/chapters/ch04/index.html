船载感知信息采集

自身状态、周边态势、运行状态、质量标注与多源互补

CHAPTER 04

船载感知信息采集

船联网的数据质量决定应用上限。海上环境中的遮挡、海况、电磁干扰、链路波动和设备异构，都会把传感器误差放大成态势误判。理解感知链条，不只是知道设备名称，更要知道数据从哪里来、在哪些环节会失真，以及系统如何把这些不确定性显式表达出来。

自身状态定位、时间基准、航向姿态、速度航程
周边态势AIS、雷达、视觉等多源互补
运行状态主机、电力、货物与舱室监测
质量意识标注、校核、同步与降级运行

GO学习目标

本章围绕“从测量到数据”的链条建立质量意识。读完后，应能解释关键感知量的误差来源、互补关系以及质量标注如何影响后续融合与决策。

理解误差

说明定位、姿态、速度等关键量的误差来源及其工程表现。

理解互补

解释 AIS、雷达、视觉等多源手段为何必须协同使用。

理解质量

说明质量标注、时间同步和在线校核为何是感知系统基础能力。

4.1船舶自身状态感知

船舶自身状态感知以位置、时间、航向、姿态、速度和航程为主。它们不仅影响导航显示，还会影响目标关联、航迹推算、自动控制和能效计算。

定位与时间基准

经纬度本身不足以解释定位可靠性，坐标基准、解状态、授时状态和时间偏差同样属于定位数据的一部分。

航向与姿态

罗经、卫星罗经、IMU 和组合导航共同决定航向与姿态质量，误差常以系统性偏差和时间错配形式出现。

速度与航程

对地速度、对水速度和航程累积值适用于不同任务，若时间基准不稳，误差会随时间累积放大。

4.1A定位数据不只有经纬度

定位数据在系统中通常以经纬度和高度出现，但真正影响可解释性的还包括坐标基准、定位解状态、精度线索和时间基准状态。

字段类别 典型内容 工程意义
位置字段 经纬度、高度 描述目标所在位置
坐标基准 WGS 84 等 决定跨系统交换时轨迹是否会整体偏移
精度与完整性线索 HDOP、完整性指标、解状态 比位置值本身更早暴露异常风险
时间字段 采集时间、到达时间、授时状态 决定多源数据能否对齐

4.1BGNSS 可用性并不是恒定的

海上环境通常有利于卫星信号接收，但遮挡、多路径、电离层扰动以及干扰都会引入误差。误差既表现为数值偏差，也表现为更新间隔变化、短时跳变和质量指示异常。

遮挡与多路径

会导致星数下降、定位短时漂移或跳变，尤其影响港区和近岸复杂环境。

电离层与大气扰动

会造成误差缓慢变化和质量指示不稳定，需依靠多系统、多频和差分增强缓解。

干扰与欺骗

会使位置或时间基准突然异常，系统必须保留状态字段并触发告警与降级。

4.1C航向姿态错误往往不是异常值，而是系统性偏差

磁罗经、陀螺罗经、卫星罗经和 IMU 的误差来源不同。姿态数据进入融合链条后，经常不会表现为明显超界，而是以时间错配、安装偏差、海况扰动和平滑掩盖的形式出现。

典型误差线索

  • 磁罗经：受磁干扰，长期偏差明显
  • 陀螺罗经：受校准与机械状态影响
  • 卫星罗经：天线基线与可见卫星影响精度
  • IMU：海况下高频波动显著

工程处理思路

保留采样频率、滤波状态和质量标记，并建立交叉校核关系，例如航向变化是否与转向率一致、姿态波动是否与海况一致。

4.2周边交通态势感知

周边态势感知的核心是识别附近目标并估计其运动趋势。单一手段很难在所有海况和能见度条件下稳定工作，因此多源互补是基本思路，而不是可选增强。

AIS

提供身份、位置、航向航速和航次信息，是合作式感知来源。

雷达

提供独立位置观测和全天候探测能力，是非合作式感知核心手段。

视觉

提供近距离外观、类别和行为细节，是确认与分类的重要补充。

4.2A多源感知覆盖范围图

AIS、雷达和视觉在覆盖距离、探测方式和语义丰富度上各不相同。系统的可靠性来自互补，而不是来自某一种来源单独足够强。

AIS 提供身份和意图线索，雷达提供独立位置与速度观测，视觉提供近距离确认和辅助分类。交叉校核区是多源互补最有价值的区域。

4.2BAIS 的作用和边界要一起理解

AIS 在船联网中既是数据来源，也是一种跨主体协同机制。理解 AIS 的关键，不在于背报文编号，而在于看清数据生成链条、时效边界和不同接收路径的差异。

优势

  • 可提供身份、位置、航向航速和航次信息
  • 为船船与船岸态势共享提供底座信息
  • 目标类别不同，可显式进入风险判断逻辑

边界

  • 自报属性带来缺失、延迟和伪造风险
  • 船载、岸基、卫星接收路径的覆盖与时效不同
  • 字段来源不同，可信度和校核方式也不同

4.3船舶运行状态感知

船舶运行状态感知聚焦主机、动力、电力、货物和舱室状态。与导航数据相比，这类数据更直接进入健康管理、运维决策和绿色航运核算。

主机与动力系统

转速、扭矩、油耗、推进器状态决定运行安全和能效分析基础。

机舱设备健康

温度、压力、振动与状态量共同支撑健康管理和预测性维护。

货物与舱室监测

液位、气体、温湿度和阀位等数据常直接关联安全风险。

4.3A能效指标来自多组数据协同

单位航程能耗、排放统计和工况分析都不是单个传感器的结果，而是燃油消耗、航程、速度、载荷和环境背景共同作用的结果。

消耗量

燃油流量或用油统计提供分子信息。

航程与速度

决定分母与运行背景。

工况

负荷、推进状态影响解释口径。

环境背景

风浪流与气象海况帮助解释波动。

4.4数据质量及标注

质量标注的意义，不是简单给数据贴上“好/坏”标签，而是把有效性、来源、时间语义和处理链路信息显式写进数据，使后续模块能够做出可解释取舍。

典型质量问题形态

  • 缺测、漂移、延迟、异常值
  • 卡死值、量化误差、乱序、重复
  • 跨源不一致、单位或映射错误

最小质量标注集合

  • 有效性状态
  • 来源标识与接口链路版本
  • 采集时间、到达时间、时效估计
  • 异常事件编号或处理动作记录

4.4A质量问题如何被发现

质量问题不需要复杂算法才能发现。持续监测采样间隔、时间戳、变化率和跨源差异，就能在感知侧提前发现大量问题。

范围检查

适合发现超出物理边界和明显不可能值。

变化率检查

适合发现跳变、抖动和短时异常冲击。

一致性检查

适合发现多源之间的长期偏离和身份错配。

4.5多源互补与时间同步

多源互补的目标不是简单叠加更多数据，而是在不确定环境下形成最小可靠组合。时间同步则是所有互补和融合的前提条件。

互补关系

AIS 提供身份语义，雷达提供独立观测，视觉提供目标确认；GNSS、IMU、罗经共同支撑自身状态连续性。

时间同步

即使各传感器误差都很小，只要时间轴不一致，融合结果就会出现系统性错配。

4.5A融合前先做前置处理

在进入融合前，数据通常需要完成坐标统一、单位统一、字段映射、异常过滤和延迟估计。否则融合面对的输入语义并不一致，结果就很难稳定。

坐标与安装几何

不同传感器观测需转换到统一坐标系。

延迟估计

AIS、雷达、视觉的时效差异需要被显式估计。

目标关联

关联结果本身也应作为可解释数据产品输出。

4.5B弱网断链下的感知闭环

弱网和断链在海上是常态。感知闭环要求关键感知数据和告警逻辑尽量放在船端，使驾驶台告警、关键设备保护和事件记录不依赖岸端服务存在。

断链期间必须保留的能力

  • 关键态势感知与告警
  • 关键设备保护
  • 事件与质量日志记录

岸端的角色

岸端提供增强分析、回放、运维和跨船协同能力，但不应成为船端最低安全能力的唯一依赖。
