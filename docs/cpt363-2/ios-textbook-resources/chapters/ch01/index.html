物联网技术基础

概念起源、传感器网络、嵌入式平台、云边协同、船联网衔接

章节导览

这一章对应教材第一章的完整主线，不只是“物联网是什么”，而是回答五个连续问题：物联网为什么出现、数据从哪里来、设备如何长期运行、数据应该在哪里处理、这些通用机制为什么在船联网场景下会变得更严格。

建议把本页当作线上版讲义浏览：先看“章节路线图”，再按 1.1 到 1.5 顺序学习，最后用“术语速览”和“思考题”完成收束。

学习目标

本章用物联网的通用技术链条作为共同语言，为后续船联网内容建立可复用的认识框架。学完后，学生不应只会复述名词，而应能判断一个方案为什么可行、为什么会失效、以及它迁移到船联网后哪些约束会被放大。

  • 能够解释物联网概念起源、核心定义和四个基本特征
  • 能够用感知、网络、平台、应用四层描述物联网系统的数据闭环
  • 能够说明传感器网络、嵌入式平台、物联网操作系统各自解决的问题
  • 能够判断云计算和边缘计算的职责分工，并解释断网运行与数据分级的必要性
  • 能够把陆上物联网的一般规律映射到船联网场景，指出海上通信、运维和安全约束

章节路线图

1.1 概念发展

从 RFID 与自动识别出发，理解物联网的定义、特征、关键节点和典型应用。

1.2 传感器网络

明确数据从哪里来，理解采样、信号处理、误差控制和多节点协同。

1.3 嵌入式平台

理解设备侧如何运行，包括硬件平台、操作系统、中间件、时间同步和远程运维。

1.4 云边协同

理解数据在哪里处理，为什么边缘侧在受限场景下必须承担更重职责。

1.5 向船联网衔接

把本章通用机制放到海上环境中，理解为什么船联网不能直接套用陆上物联网架构。

1.1 物联网概念发展

教材第一章首先要解决“物联网到底是什么”。这里最重要的不是背定义，而是理解它为什么会从供应链识别发展成一个覆盖感知、连接、计算和协同的完整技术体系。

1.1.1 概念起源与核心定义

1999 年 MIT Auto-ID Center 在 RFID 供应链研究中明确提出“物联网”术语，最早强调的是“物品可识别、可追踪”。随后这个概念不断扩展，今天更常用的定义是：通过传感器、通信网络和计算平台，把物理世界中的对象、环境和过程转化为可采集、可传输、可处理的数据，并在此基础上实现监测、控制、优化和协同。

  • 感知能力：系统能得到位置、温度、压力、速度、状态等外部世界信息
  • 连接能力：这些信息能在设备、网络和平台之间可靠传递
  • 计算能力：原始数据可以被加工成信息、知识和决策依据

1.1.2 四个基本特征

教材把物联网的能力链条归纳为四个环节。它们不是平行罗列关系，而是彼此依赖的工程链条。

特征 教材含义 工程提醒
全面感知 持续获取对象、环境、过程状态 没有可靠感知，后续分析只是放大误差
可靠传输 在带宽、覆盖和成本约束下送达关键数据 关键数据优先、缓存补传、断链降级是常见策略
智能处理 清洗、汇总、融合、告警、预测 算力位置决定时延、成本和可用性
应用服务 把结果交给人或系统使用 展示、告警、控制和共享都属于服务输出

1.1.3 发展关键节点与典型领域

教材把发展历程拆成概念萌芽、技术成熟和规模应用三个阶段。观察这条时间线，学生应看到两条主线：一条是技术能力增强，另一条是应用范围扩展。

  • 1990s 中后期至 2005：RFID、传感器网络，重点解决识别和基础采集
  • 2005 至 2015：移动通信、嵌入式系统成本下降、云平台兴起，应用开始成规模
  • 2015 至今：云计算、大数据、人工智能与边缘计算深度融合，形成系统化解决方案

智慧城市、智能制造、智慧农业、智慧医疗、智慧物流等行业虽然目标不同，但本质上都遵循相同逻辑：通过感知获取数据，通过传输汇聚数据，通过处理挖掘价值，通过应用实现目标。

1.1.4 系统要素与数据链条

教材把物联网工程系统抽象为感知层、网络层、平台层和应用层。这个四层结构特别适合在线上教学中做“总图”，因为它同时解释了系统边界和数据闭环。

  • 感知层：负责采样、识别、本地预处理，是数据入口
  • 网络层：负责接入、路由、传输，是数据通道
  • 平台层：负责存储、计算、规则和服务编排，是数据加工中心
  • 应用层：负责展示、告警、优化和控制，是数据出口

关键点在于：数据链条不是单向上报。应用层的判断会反过来影响采样频率、传输优先级和边缘侧策略，因此物联网更像“可自适应的闭环系统”，而不是静态数据采集通道。

1.2 传感器网络

教材第二条主线回答“数据从哪里来”。传感器网络是物联网感知层的核心基础，它决定系统能否在空间上覆盖目标、在时间上持续观测目标。

1.2.1 传感器网络的基本概念

传感器网络由大量分布式节点、汇聚节点和管理平台组成。节点负责采样和短距离通信，汇聚节点负责汇总和上行，管理平台负责长期存储、分析和应用服务。它的典型特点是：节点多、单节点能力弱、供电受限、带宽受限、拓扑可能动态变化。

  • 节点数量大，但算力、存储和电池容量有限
  • 无线链路易受干扰，多跳转发会形成瓶颈节点
  • 系统设计必须关注能耗、容错和负载均衡

1.2.2 传感器工作原理与选型

教材强调，传感器是物理量到数字信号的入口。温度、压力、位置、惯性、图像等不同对象，需要不同的传感器和不同的信号转换原理。学生在这里应建立一个基本判断：没有“万能传感器”，只有按测量对象、环境条件和成本约束做出的选型结果。

  • 温度、湿度、压力等环境量：关注量程、精度和长期稳定性
  • 定位和惯性类传感器：关注采样频率、精度和时间同步
  • 视觉类传感器：关注分辨率、帧率、光照条件和带宽压力

1.2.3 数据采集、信号处理与误差控制

教材这里很关键，因为它把“数据并非天然可信”说得很清楚。采样频率过低会丢失快速变化，过高又会增加带宽和存储负担；滤波、校准和特征提取是提高可用性的常见手段；范围检查、一致性检查和趋势检查则是最基础的质量控制方法。

课堂提示

可以让学生把“传感器数据质量问题”分成三类记忆：一类来自传感器本体误差，一类来自采集与时钟问题，一类来自传输与缓存问题。后续船联网的质量标注、本地缓存和数据治理都从这里展开。

1.3 嵌入式平台与物联网操作系统

教材第三条主线回答“设备如何可靠运行”。物联网不是只有云平台，真正长期暴露在现场环境中的，是大量边缘设备、网关和嵌入式终端。

1.3.1 嵌入式平台的角色

嵌入式平台承担感知、控制和边缘计算任务，通常资源比通用计算机更受限，因此必须在功耗、实时性、稳定性和可维护性之间做平衡。它们不是“缩小版 PC”，而是面向特定任务设计的专用运行环境。

1.3.2 物联网操作系统的功能边界

教材把嵌入式 Linux、FreeRTOS、Zephyr、LiteOS 等系统拿来对比，目的不是要求背型号，而是理解它们解决的问题不同：

  • 轻量级 RTOS 更强调实时调度、低资源占用，适合传感器节点
  • 嵌入式 Linux 更适合运行复杂应用和多协议网关
  • 中间件位于操作系统和业务应用之间，负责协议、安全、数据封装等跨应用能力

1.3.3 软件分层、时间同步与远程运维

教材把设备软件从底到上拆成硬件抽象层、驱动层、操作系统层、中间件层和应用层。这个分层最重要的价值是“隔离变化”：更换传感器主要影响驱动层，切换协议主要影响中间件层，而不应该把所有改动扩散到整个系统。

  • 时间同步：NTP、PTP、GNSS 授时帮助多设备数据对齐，时间状态本身也应被记录
  • 状态监测：CPU、内存、通信质量、传感器状态等指标支撑远程诊断
  • OTA 升级：升级包签名、断点续传、回滚机制、A/B 分区是可靠升级关键

1.4 云计算与边缘计算

教材第四条主线回答“数据在哪里处理”。这部分是后续船联网学习的关键，因为海上场景会把云边协同的取舍推到更极端的位置。

1.4.1 云计算提供什么

云计算提供弹性算力、海量存储、平台能力和统一设备管理，非常适合长周期历史数据保存、复杂分析、跨区域协同和应用托管。物联网平台常位于 PaaS 层，提供设备认证、消息路由、数据存储、规则引擎和 API 服务。

1.4.2 边缘计算为什么兴起

教材把边缘计算兴起归因于四个因素：时延要求、带宽限制、可靠性需求和隐私保护。课堂上最好把这四点变成判断题：如果任务对毫秒级响应敏感、原始数据量巨大、网络可能中断、或数据不宜全部上云，那么边缘处理就不是优化项，而是基础项。

1.4.3 云边协同、断网运行与数据分级

云和边缘不是替代关系，而是协同关系。教材给出的基本原则非常清晰：

  • 时延敏感和安全关键任务放在边缘
  • 计算密集和全局优化任务放在云端
  • 边缘侧负责预处理、缓存、本地告警和断网保底
  • 云端负责长期归档、复杂分析、跨域管理和应用托管

断网运行与数据分级是这一节最值得带到船联网的两个概念。关键告警应本地即时处理，运行状态可降频汇总，高频原始数据不必完整缓存，异常时段详细记录应优先保留。

1.5 从物联网到船联网

教材最后把本章内容往船联网场景收束。学生应在这里建立一个核心判断：船联网不是一个新发明，而是物联网在“高受限、高风险、难运维”行业中的特化形态。

1.5.1 行业适配与海上约束

不同行业的约束不同，会改变技术优先级。相比智慧城市、制造、农业和车联网，海上物联网的通信条件最受限、运维窗口最稀缺、失效后果最敏感，因此对边缘能力、远程运维和可靠性设计要求最高。

  • 通信受限：远洋主要依赖高时延、高费用、低带宽卫星链路
  • 运维困难：航次长、现场支持少，设备要长期无人值守
  • 安全敏感：错误信息可能直接影响航行判断和应急处置

1.5.2 对后续船联网学习的衔接

这一章建立的知识基础会在后续章节逐步展开：第二章解释概念边界，第三章解释体系架构，第四章进入船舶感知系统，第五章进入海上通信，第六章进入数据治理，第七章进入安全与可靠运行，第八章进入典型应用。也就是说，本章是后续七章的总入口。

术语速览

  • IoT：Internet of Things，物联网
  • WSN：Wireless Sensor Network，传感器网络
  • RFID：Radio Frequency Identification，射频识别
  • RTOS：Real-Time Operating System，实时操作系统
  • OTA：Over-The-Air，空中升级/远程升级
  • NTP / PTP：常见时间同步机制
  • Cloud-Edge Collaboration：云边协同

思考题

  1. 物联网四个基本特征之间有什么依赖关系？如果传输环节不可靠，会怎样影响感知、处理和应用结果？
  2. 为什么物联网设备通常不直接使用通用操作系统？RTOS 和嵌入式 Linux 各适合什么场景？
  3. 请为一个实时监测系统划分云端和边缘端职责，并说明你的判断依据。
  4. 当网络中断时，关键告警、运行状态、高频原始数据和运维日志应分别如何处理？
  5. 结合本章内容，说明船联网相比陆上物联网多了哪些特殊约束，这些约束会如何改变架构设计。
