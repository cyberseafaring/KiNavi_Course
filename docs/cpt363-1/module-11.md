# Workshop 3 AIS / ECDIS 融合训练

本训练用于第6章或第8章之后，帮助学生比较雷达、AIS 和 ECDIS 信息的一致性，理解综合显示并不能替代人工复核。

## 训练目标

- 比较雷达目标、AIS 目标和电子海图对象的来源差异。
- 识别目标位置、航向、航速、名称和状态不一致的情况。
- 说明传感器延迟、输入错误和目标关联失败带来的风险。
- 形成驾驶台多源信息交叉验证步骤。

## 训练流程

1. 给定一个包含雷达目标、AIS 目标和海图对象的场景。
2. 检查目标位置、航向、航速、CPA / TCPA 和目标名称。
3. 标出一致、不一致和需要继续观察的信息。
4. 说明哪些信息可用于避碰判断，哪些只作为辅助参考。
5. 给出人工复核和记录建议。

## 交付物

提交一张“雷达-AIS-ECDIS 一致性检查表”，至少列出 5 个检查项和对应风险。

## 一致性检查表

| 检查项 | 雷达侧 | AIS / ECDIS 侧 | 判断 |
| --- | --- | --- | --- |
| 位置 | 回波位置、距离、方位 | AIS 坐标、海图显示 | 是否有明显偏移 |
| 航向 | 真矢量方向 | AIS COG / heading | 是否同向或存在延迟 |
| 航速 | ARPA 计算速度 | AIS SOG / STW | 是否与场景相符 |
| 身份 | 雷达目标编号 | MMSI、船名、类型 | 是否关联到正确目标 |
| 风险 | CPA / TCPA | AIS CPA / TCPA 或 ECDIS 警戒 | 是否给出一致风险等级 |

## 课堂任务

1. 给出一个包含三个目标的场景，让学生标出可靠、一致性不足和需继续观察的目标。
2. 让学生判断 AIS 目标与雷达目标错配时，避碰判断应优先使用哪些信息。
3. 设计一次驾驶台交叉验证口令，用中文说明判断，用英文说出关键术语。

## 评价维度

| 维度 | 达标表现 |
| --- | --- |
| 来源意识 | 能区分直接观测、广播数据和海图数据 |
| 风险处理 | 不把 AIS 名称或颜色直接等同于安全结论 |
| 复核流程 | 有持续观测、目标关联和人工确认步骤 |
| 双语表达 | 能准确使用 association, target report, radar overlay 等术语 |
