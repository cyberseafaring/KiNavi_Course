# Workshop 4 故障与标准阅读

本训练用于第7章之后，帮助学生把设备异常、维护检查和标准要求放在同一个安全管理框架中分析。

## 训练目标

- 从故障现象推断可能原因。
- 设计逐步检查路径，避免直接跳到单一结论。
- 区分立即影响航行安全的问题和可计划维护的问题。
- 用规范语言记录缺陷、限制和复核结果。

## 训练流程

1. 选择一个故障现象，例如无回波、目标拖尾、方位偏差、距离误差或 ARPA 跟踪不稳定。
2. 列出可能原因，覆盖天线、发射接收、传感器输入、显示设置和外部环境。
3. 设计检查顺序，说明每一步希望排除什么。
4. 判断是否需要改变航行计划、加强瞭望或限制设备使用。
5. 写出维护记录和课堂汇报摘要。

## 交付物

提交一张故障分析表，包含“现象、可能原因、检查步骤、航行影响、记录内容”五列。

## 故障案例库

| 案例 | 可能原因 | 首要检查 |
| --- | --- | --- |
| 无目标回波 | 发射机未正常工作、天线故障、增益过低、量程不当 | 电源、发射状态、增益、天线旋转 |
| 近区大片杂波 | 海况、海杂波抑制不当、增益过高 | 量程、海浪抑制、增益 |
| 方位整体偏移 | 艏线误差、航向输入异常、天线编码问题 | 艏线、罗经输入、固定目标方位 |
| ARPA 跟踪跳变 | 目标弱、遮挡、海况差、航向或速度输入异常 | 目标稳定性、传感器输入、矢量模式 |
| 雷达叠加偏移 | GNSS、航向、ECDIS 或坐标基准问题 | 位置源、航向源、海图叠加设置 |

## 课堂任务

1. 选择一个故障案例，按“现象 -> 可能原因 -> 检查步骤 -> 航行影响 -> 记录内容”写出诊断表。
2. 说明该故障是否要求改变航行计划、增派瞭望或限制设备使用。
3. 用英文写出一条简短交接班描述，例如：`Radar bearing error suspected; verify with fixed target before using radar for precise fixing.`

## 评价维度

| 维度 | 达标表现 |
| --- | --- |
| 排查逻辑 | 原因覆盖设备、输入、环境和操作 |
| 安全意识 | 能把故障转化为航行限制和补救措施 |
| 记录质量 | 记录具体、可复核、便于交接班 |
| 标准意识 | 能联系性能要求、维护记录和船舶安全管理 |
